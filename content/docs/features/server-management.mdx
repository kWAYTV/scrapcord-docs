# Server Management

Scrapcord provides comprehensive server management features through Discord slash commands. This guide covers all aspects of managing your Rust servers.

<Callout type="warning">
  All server management commands require administrator permissions in Discord.
</Callout>

## Adding Servers

<Steps>
    <Step>
        **Start the Process**
        ```bash
        /server add
        ```
        This will open a modal form with the following fields.
    </Step>
    <Step>
        **Enter Server ID**
        - Must be alphanumeric characters, underscores, or hyphens
        - Used as unique identifier
        - Example: `rust-main` or `staging_server`
        
        <Callout type="error">
          Server IDs cannot be changed after creation. Choose carefully!
        </Callout>
    </Step>
    <Step>
        **Enter Server Name**
        - Display name for the server
        - Can contain spaces and special characters
        - Example: `üåü Main Server [Monthly]`
        
        <Callout type="note">
          This is the name that will be displayed in commands and logs.
        </Callout>
    </Step>
    <Step>
        **Specify Config Name**
        - Optional configuration name
        - Defaults to "default" if not specified
        - Example: `monthly` or `weekly`
        
        <Callout type="info">
          You can have multiple configurations for different wipe schedules.
        </Callout>
    </Step>
    <Step>
        **Configure Server**
        After adding the server, you'll receive a template configuration file. Modify it according to your needs and upload using:
        ```bash
        /config add server:rust-main file:config.yaml name:default
        ```
    </Step>
</Steps>

## Server Commands

### Basic Management

<Steps>
    <Step>
        **List Servers**
        ```bash
        /server list
        ```
        Shows all registered servers with:
        - Server ID and name
        - Creation date
        - Available configurations
    </Step>
    <Step>
        **Delete Server**
        ```bash
        /server delete server:rust-main
        ```
        <Callout type="error">
          This will delete the server and ALL its configurations permanently!
        </Callout>
    </Step>
</Steps>

### Server Actions

<Steps>
    <Step>
        **Execute Commands**
        <Tabs items={["Basic", "Advanced", "Examples"]}>
            <Tab>
                ```bash
                /action command server:rust-main command:"say Hello"
                ```
            </Tab>
            <Tab>
                ```bash
                /action command server:rust-main command:"oxide.reload *" config:weekly
                ```
            </Tab>
            <Tab>
                ```bash
                # Broadcast message
                /action command server:rust-main command:"say Server restart in 5 minutes"
                
                # Reload plugin
                /action command server:rust-main command:"oxide.reload BetterLoot"
                
                # Set time
                /action command server:rust-main command:"env.time 12"
                ```
            </Tab>
        </Tabs>
        
        <Callout type="note">
          Commands are executed exactly as entered. Check your syntax carefully!
        </Callout>
    </Step>
    <Step>
        **Send Chat Messages**
        <Tabs items={["Basic", "Advanced", "Examples"]}>

</Steps>

## Configuration Management

### Managing Configs

<Steps>
    <Step>
        **Add Configuration**
        ```bash
        /config add server:rust-main file:config.yaml name:weekly
        ```
        
        <Callout type="note">
          The configuration file must be in YAML format.
        </Callout>
    </Step>
    <Step>
        **List Configurations**
        ```bash
        /config list server:rust-main
        ```
        Shows all configurations for a server with:
        - Configuration name
        - Key settings
        - File patterns
        - Command counts
    </Step>
    <Step>
        **View Configuration**
        ```bash
        /config show server:rust-main name:weekly
        ```
        Displays detailed configuration including:
        - All settings
        - File patterns
        - Commands
        - World seeds
    </Step>
    <Step>
        **Delete Configuration**
        ```bash
        /config delete server:rust-main name:weekly
        ```
        
        <Callout type="warning">
          The default configuration cannot be deleted while other configurations exist.
        </Callout>
    </Step>
</Steps>

## Permissions

<Steps>
    <Step>
        **Discord Permissions**
        - Administrator role in Discord
        - Access to command channels
        - View server logs permission
    </Step>
    <Step>
        **Pterodactyl Permissions**
        - Valid API key
        - Server access
        - Command execution rights
        - File management access
    </Step>
    <Step>
        **Server Requirements**
        - RCON enabled
        - Valid connection details
        - Proper plugin configuration
    </Step>
</Steps>

  For more information about automated wipes and scheduling, see the [Automated Wipes](/docs/features/automated-wipes) guide.
</Callout>
